# Code Of Conduct

SkillBox contributors are expected to keep project spaces practical, respectful,
and focused on improving the software.

## Expected Behavior

- Be direct and constructive.
- Assume good intent while still checking facts.
- Keep bug reports and reviews grounded in reproducible evidence.
- Respect privacy when discussing local paths, skill contents, logs, or security
  reports.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Publishing private user data, local file contents, credentials, or exploit
  details without permission.
- Repeatedly derailing technical discussions after maintainers have redirected
  the thread.

## Enforcement

Maintainers may edit or remove comments, lock discussions, block accounts, or
decline contributions that violate this code. Security-sensitive issues should
follow [SECURITY.md](SECURITY.md).
